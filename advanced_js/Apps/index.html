Queuing API: Proof of Concept Samples

  • About this API
  • Using the asynchronous features and API of javascript, this is an 'iteratively building' API to create a queue system, where it would, typically, be applied to a despatch system like a pizza delivery service or even an ambulance service The library sits in the file: queueLib.js : which has the API to starts, run and update (including addig and deleting) a single set of jobs
  • About the sample code
  • Currently two applications, using a duration to run one and two instances of the Queueing object
  • Conceptually a list of elements run until a status, then get moved to another queue. The process runs for a configurable duration
  • Single instance usade - delivery queue
  • On the left side is a set of executing jobs, each shown in green background.
  • Once each job gets over it gets transferred to the right, availabe queue, and displays in a pink background
  • Available jobs are restarted and get pushed to the executing queue
  • Two instance usade - orders processing and delivery
  • Processing takes place in the right colunn and progress of processing shows as % age completion
  • After processing, the order's pushed to the central column, going to 'delivering' status (% age completion)
  • Once delivered, the order is moved to the delivered and complete list
The focus has been, so far, to showcase the functionality of the API more than styling and positioning, confessing to be a little lazy and strapped for time. The examples are, at best, 'stylistically' challenged. Please bear with me
  • API quick view:
  • initialize(list): Initializes the queue with a list of job objects
  • pollQueue(timer, job) Starts the polling for a job.. wrapper around a closure function which starts the job and monitors the end condition
  • setRandomInterval(callback, rules) Executes the callback function specified at intervals specified as per the rules
  • startJobs() Runs the polling for the list of job objects (internally calls pollQueue)
  • stopAgents() Uses window.clearInterval to stop the running jobs .
  • abort() Suspends the run.
  • addJob(job, index=-1) Adds a job. pauses the run to pick up the new element into the queue, and then resume. index can be zero (job added at the start), positive value (job added at that position in the queue) or default -1 (job appended)
  • deleteJob(job, index=-1) Remove a job. pauses the run to remove the element from the queue, and then resume. index usage same as above
  • Function Handles (to be provided by the user)
    • displayInitialize(job): Called when adding a job
    • displayRunStatus(job): Called when the job is running
    • displayEndStatus(job): Called when the job is ended
